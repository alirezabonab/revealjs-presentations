# Intake Checklist

Use this checklist before building or changing presentations.

## Ask these questions when missing

- What is the goal of this presentation?
- Who is the audience?
- Is this a new presentation or an update?
- What route or presentation slug should be used?
- How many slides, or how long should the talk be?
- What tone should it have: formal, technical, bold, simple, sales, calm?
- What colors, brand rules, or style direction should be used?
- What source material should be used: notes, docs, links, screenshots, images?
- Should speaker notes be added?
- Are there deadline or format constraints?

## Minimum brief before building

Try to get at least these five items:

- goal
- audience
- new vs existing presentation
- presentation slug or target deck
- preferred visual direction

## Build choices after answers

- If the user wants a new deck, create a new presentation folder.
- If the user wants extra content for one deck, add or update markdown files inside that presentation.
- If the user wants a new look, update `assets/theme.css` first.
- If the user gives brand assets, keep them local to that presentation folder.
